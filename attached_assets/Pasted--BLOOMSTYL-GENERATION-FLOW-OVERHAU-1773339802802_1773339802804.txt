PART 1 — FIX THE GENERATION FREEZE (CRITICAL — DO FIRST)
═══════════════════════════════════════════════════════════════

The generate button currently loops and freezes.
Find and fix all of the following likely causes:

── DIAGNOSE AND FIX ──────────────────────────────────────────

1. ASYNC/AWAIT DEADLOCK
   Check every async function in the generation pipeline.
   Ensure no await is called inside a non-async function.
   Ensure no Promise is left unresolved or uncaught.
   Add try/catch around every API call:

   async function generateWorksheet(params) {
     try {
       const result = await callClaudeAPI(params);
       return result;
     } catch (error) {
       console.error('Generation failed:', error);
       setGenerationState('error');
       showErrorMessage(error.message);
       return null;
     }
   }

2. INFINITE RENDER LOOP
   Check all useEffect / reactive state watchers.
   Any state update that triggers a re-render that 
   triggers another state update causes infinite loop.
   
   Fix pattern: add proper dependency arrays.
   Never update state inside a watcher that watches 
   that same state.
   
   Add a generation lock:
   let isGenerating = false;
   
   function handleGenerate() {
     if (isGenerating) return;  // prevent double-fire
     isGenerating = true;
     generateWorksheet()
       .finally(() => { isGenerating = false; });
   }

3. API CALL NEVER RESOLVING
   Add a timeout to every Claude API call:
   
   const controller = new AbortController();
   const timeout = setTimeout(() => {
     controller.abort();
   }, 30000); // 30 second max
   
   fetch(API_URL, { 
     signal: controller.signal,
     ...options 
   }).finally(() => clearTimeout(timeout));
   
   If the call times out, show:
   "Taking longer than expected — 
   [Try Again] [Simplify Request]"

4. JSON PARSE FAILURE CAUSING LOOP
   If Claude returns malformed JSON, the parse fails,
   the app retries, gets same bad JSON, loops forever.
   
   Fix: validate JSON before parsing.
   If invalid, do NOT retry automatically.
   Show error state with manual retry button.
   
   function safeParseJSON(str) {
     try {
       // Strip markdown code fences if present
       const cleaned = str
         .replace(/^```json\n?/, '')
         .replace(/\n?```$/, '')
         .trim();
       return JSON.parse(cleaned);
     } catch(e) {
       console.error('JSON parse failed:', e);
       return null;
     }
   }

5. BUTTON NOT DISABLED DURING GENERATION
   The generate button must be disabled immediately 
   on click and re-enabled only when generation 
   completes or errors.
   
   generateBtn.disabled = true;
   generateBtn.textContent = 'Generating...';
   try {
     await generate();
   } finally {
     generateBtn.disabled = false;
     generateBtn.textContent = 'Generate';
   }

6. STATE NOT RESETTING BETWEEN ATTEMPTS
   If a previous failed generation left state dirty,
   the next attempt breaks.
   
   Always reset generation state before starting:
   function resetGenerationState() {
     setLayouts([]);
     setError(null);
     setProgress(0);
     setGenerationPhase('idle');
   }

After fixes are in place, add a console log at each
major step of the generation pipeline so failures are 
immediately visible:
   console.log('[GEN] Step 1: Parsing prompt...');
   console.log('[GEN] Step 2: Calling Claude API...');
   console.log('[GEN] Step 3: Received response...');
   console.log('[GEN] Step 4: Rendering layouts...');

PART 3 — PARALLEL JOB ARCHITECTURE
═══════════════════════════════════════════════════════════════

The app must run multiple jobs simultaneously.
Nothing should block the UI thread.

── JOB QUEUE SYSTEM ──────────────────────────────────────────

Implement a lightweight job queue:

const JobQueue = {
  jobs: [],
  running: 0,
  maxConcurrent: 3,

  add(jobFn, priority = 'normal') {
    return new Promise((resolve, reject) => {
      this.jobs.push({ jobFn, resolve, reject, priority });
      this.jobs.sort((a, b) => 
        a.priority === 'high' ? -1 : 1
      );
      this.flush();
    });
  },

  async flush() {
    while (this.running < this.maxConcurrent 
           && this.jobs.length > 0) {
      const job = this.jobs.shift();
      this.running++;
      try {
        const result = await job.jobFn();
        job.resolve(result);
      } catch(e) {
        job.reject(e);
      } finally {
        this.running--;
        this.flush();
      }
    }
  }
};

JOBS THAT RUN IN PARALLEL:

Job 1 (high priority): Generate Layout A content
Job 2 (high priority): Generate Layout B content  
Job 3 (high priority): Generate Layout C content
Job 4 (normal): Pre-load font assets for selected style
Job 5 (normal): Pre-fetch relevant clipart/images
Job 6 (low): Pre-warm differentiation if selected

Results from Jobs 1–3 render independently as they 
complete. Teacher sees Layout A appear, then B, then C.
Not all at once after a long wait.
